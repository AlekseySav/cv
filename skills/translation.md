# Translation rules

## translate following words as given:
- skills -> навыки
- experience -> опыт работы
- projects -> проекты
- applied science -> прикладная математика и информатика
- engineer, software engineer -> разработчик, senior -> старший
- Within skills, keep only english language skill (i.e. Spoken: ... -> English: B2/C1)

## not to be translated:
- "Remote", "Hybrid", "On-Cite"
- "full-time", "part-time"
- "Tech lead"
- company names, technology namings
- techical terms without well-established translation
- concept skills

## prefer:
- intuitive word-use over direct one-to-one translation
- terms over regular words (e.g. неконсистентный is better than несогласованный, fuzzy-поиск is better than нечёткий поиск)
- terse technical notation over spelled-out phrasing (e.g. "4x быстрее" not "в 4 раза быстрее")
- avoid repeating the same verb twice in one bullet even if the literal EN translation would (pick a synonym or restructure)

## length parity:
- Cyrillic text runs longer than English for the same meaning. Each translated bullet/line should occupy
  roughly the same visual space as its EN counterpart (line count, wrap points) so the ru CV fits the same
  page count as the en CV.
- when a literal translation would overflow, cut filler words first (опыта в, количество, для), not content
- do not use "→"/">" as compact separators -- disallowed by cv.schema.json plaintext pattern
